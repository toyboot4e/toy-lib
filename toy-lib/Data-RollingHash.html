Data.RollingHash

Description

The rolling hash algorithm lets you create fastly ((O(1))) comparable / concatanatable string slice in after \(O(N)\) preparation.

I suspect if slices longer than the orignal string can be calculated without panic in my implementation.

data RollingHash b p Source #

Rolling hash of a string.

Internals

Slice (2, 4) of "abcdef" is given as this:

           s :=     a       b       c       d       e
           s4 = b^4 a + b^3 b + b^2 c + b^1 d + b^0 e
           s2 = b^1 a + b^0 b
s4 - s2 * b^3 =                 b^2 c + b^1 d + b^0 e
